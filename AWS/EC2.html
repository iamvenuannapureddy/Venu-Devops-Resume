AWS Elastic Compute Cloud (EC2)

AWS EC2 is a web service that provides secure, resizable compute capacity in the cloud.

Key Concepts of AWS EC2

  • Instances: Virtual machines in AWS EC2.
  • AMIs (Amazon Machine Images): Pre-configured templates for instances.
  • Instance Types: Different hardware configurations.
  • Security Groups: Firewall rules controlling inbound and outbound traffic.
  • Elastic IPs: Static public IP addresses for instances.
  • Key Pairs: Used for SSH authentication.
  • Volumes & Snapshots: Persistent storage and backup for instances.

How to Launch and Manage EC2 Instances

Step 1: Launch an EC2 Instance

  1. Sign in to the AWS Management Console.
  2. Go to EC2 Dashboard and click Launch Instance.
  3. Select an Amazon Machine Image (AMI).
  4. Choose an Instance Type (e.g., t2.micro for free tier).
  5. Configure instance details, add storage, and set up security groups.
  6. Review and click Launch.

Step 2: Connect to an EC2 Instance

  1. Go to EC2 Dashboard > Instances.
  2. Select your instance and click Connect.
  3. Copy the SSH command and execute it in a terminal:
    ssh -i key.pem ec2-user@your-instance-ip

Step 3: Assign an Elastic IP

  1. Go to EC2 Dashboard > Elastic IPs.
  2. Click Allocate Elastic IP and assign it to your instance.

Step 4: Create a Security Group

  1. Go to EC2 Dashboard > Security Groups.
  2. Click Create Security Group.
  3. Define rules for inbound and outbound traffic.
  4. Attach the security group to your instance.

Step 5: Create a Snapshot

  1. Go to EC2 Dashboard > Volumes.
  2. Select a volume and click Create Snapshot.
  3. Use snapshots for backup or to create new volumes.

How to Use EC2

  • Web Hosting: Run websites and applications.
  • Development & Testing: Spin up temporary instances.
  • Data Processing: Run batch jobs and analytics.
  • Scaling: Use Auto Scaling Groups (ASG) to handle traffic spikes.
  • Security: Implement IAM roles and security groups.
