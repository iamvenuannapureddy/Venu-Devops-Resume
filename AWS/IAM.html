AWS Identity and Access Management (IAM)

AWS IAM is a web service that enables secure control of AWS services and resources.

Key Concepts of AWS IAM

  • Users: Individuals or applications requiring AWS access.
  • Groups: Collections of users for easier permission management.
  • Roles: Assumed temporarily by authorized entities.
  • Policies: JSON-based permissions for users, groups, and roles.
  • MFA: Adds an extra security layer.
  • Access Keys: Used for programmatic access via CLI or SDK.

How to Create and Use IAM

Step 1: Create an IAM User

  1. Sign in to the AWS Management Console.
  2. Go to IAM Console > Users > Add User.
  3. Set username and access type.
  4. Attach policies or add to a group.
  5. Review and create the user.

Step 2: Create an IAM Group

  1. Navigate to IAM Console > Groups > Create Group.
  2. Set a name and attach policies.
  3. Add users to the group.
  4. Click Create.

Step 3: Create an IAM Role

  1. Go to IAM Console > Roles > Create Role.
  2. Select a trusted entity (AWS service, another AWS account, etc.).
  3. Attach permissions policies.
  4. Review and create the role.

Step 4: Attach a Policy to a User/Group/Role

  1. Navigate to Users or Groups.
  2. Go to Permissions and click Add Permissions.
  3. Attach an AWS Managed or Custom Policy.

Step 5: Enforce MFA

  1. Go to IAM Console > Users.
  2. Select a user and click Manage MFA.
  3. Assign an MFA device.

How to Use IAM

  • Authentication: Use passwords or access keys to log in.
  • Authorization: Attach policies to define access rights.
  • Cross-Account Access: IAM roles allow secure access across accounts.
  • Temporary Access: Use roles instead of long-term credentials.
  • Automation: Integrate IAM with AWS SDKs and CLI.
